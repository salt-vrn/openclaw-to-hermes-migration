# OpenClaw → Hermes Migration

🌐 [English](README.md)

Пошаговый скилл для миграции AI-агентов с OpenClaw на [Hermes Agent](https://hermes-agent.nousresearch.com/docs).

## Что это

Структурированное руководство миграции, которому LLM-агент следует при переносе клиента с OpenClaw на Hermes:

- **Анализирует** существующий профиль OpenClaw (файлы, скрипты, крон, секреты)
- **Планирует** миграцию перед тем как что-то делать
- **Создаёт** структуру workspace по [hermes-advanced-memory](https://github.com/salt-vrn/hermes-advanced-memory)
- **Мигрирует** файлы, секреты (.env), скрипты, cron-задачи, конфиг
- **Отчитывается** что сделано, что осталось, что требует ручной работы

## Почему скилл, а не скрипт?

У каждого агента своя структура OpenClaw. У кого-то 5 файлов, у кого-то 200. У кого-то свои названия папок. Скрипт не понимает контекст — LLM-агент понимает.

Скилл даёт:
- **Правила** (что НИКОГДА не делать)
- **Фазы** (что в каком порядке)
- **Проверки** (как убедиться на каждом шаге)
- **Ссылки на документацию** (где читать свежие доки)

Агент адаптируется под каждый конкретный случай.

## Установка

```bash
hermes skill install salt-vrn/openclaw-to-hermes-migration
```

Или вручную:

```bash
cd ~/.hermes/skills/
git clone https://github.com/salt-vrn/openclaw-to-hermes-migration.git
```

## Предварительные условия

Перед миграцией применить [hermes-advanced-memory](https://github.com/salt-vrn/hermes-advanced-memory) для организации целевого workspace.

## 10 фаз

1. **Анализ** — инвентаризация всех файлов в профиле OpenClaw
2. **План** — представить план миграции, дождаться утверждения
3. **Создание структуры** — построить workspace с папками проектов
4. **Миграция файлов** — распределить (НЕ копировать) MEMORY.md, TOOLS.md
5. **Миграция секретов** — credentials.json → .env
6. **Миграция скриптов** — обновить пути, заменить credentials.json на os.getenv()
7. **Миграция cron-задач** — пересоздать в Hermes, тестировать по частям
8. **Настройка Hermes** — группы, пользователи, поведение
9. **Обновление AGENTS.md** — создать индекс
10. **Отчёт** — итоги: что сделано и что осталось
